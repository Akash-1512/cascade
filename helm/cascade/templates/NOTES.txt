Auth mode: {{ .Values.api.authMode }}

{{- if eq .Values.api.authMode "dev" }}

  Dev mode: any UUID is accepted as a bearer token. Suitable for staging
  or restricted-access clusters; switch to 'jwt' for production by
  setting api.authMode=jwt and configuring secrets.jwt.* in values.yaml.

{{- else if eq .Values.api.authMode "jwt" }}

  JWT mode active. The API verifies bearer tokens against:
    Issuer:    {{ .Values.secrets.jwt.issuer }}
    Audience:  {{ .Values.secrets.jwt.audience }}
    JWKS URL:  {{ .Values.secrets.jwt.jwksUrl }}

  If any are blank, the API will return 503 for any authenticated
  request — set them or revert to authMode: dev.

{{- end }}

Postgres:
{{- if .Values.postgresql.enabled }}

  The chart installed Postgres as a subchart. Connection string is wired
  into the API and MCP deployments via DATABASE_URL.

  Run migrations against the in-cluster Postgres:

    kubectl --namespace {{ .Release.Namespace }} exec deploy/{{ include "cascade.componentName" (dict "context" . "component" "api") }} -- \
      alembic upgrade head

  Seed the demo dataset:

    kubectl --namespace {{ .Release.Namespace }} exec deploy/{{ include "cascade.componentName" (dict "context" . "component" "api") }} -- \
      python -m cascade.scripts.seed_demo --verbose

{{- else }}

  Using external Postgres at {{ .Values.externalDatabase.host }}:{{ .Values.externalDatabase.port }}.
  Make sure migrations have been applied:

    alembic upgrade head

  And the demo seed (optional):

    python -m cascade.scripts.seed_demo --verbose

{{- end }}
